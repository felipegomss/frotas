# Checklist de planejamento

- [ ] Li a EF do módulo (specs/EFs/) e os ADRs/convenções relevantes.
- [ ] Existe (ou criei) a spec da feature em specs/features/ com critérios de aceite verificáveis.
- [ ] Plano por camada definido: domain -> contracts -> api -> app.
- [ ] Defini os testes que provam cada critério de aceite (TDD: escrevo antes do código).
- [ ] Identifiquei se toca dado de tenant / auth (se sim, a revisão de segurança é obrigatória).
- [ ] Listei riscos, fora de escopo e decisões em aberto (ADR se necessário).
- [ ] Não editei código; plano apresentado e aprovado (plan mode).
