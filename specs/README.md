# Base de conhecimento — frotas (AMPARO Frota)

Documentação viva do projeto, lida pelo time e pelo Claude Code.

## Mapa
- produto/       O que é o sistema, para quem, e o faseamento comercial.
- arquitetura/   Como o sistema é construído (monorepo, stack, multi-tenant, infra, segurança).
- EFs/           Especificações funcionais por módulo (o "o quê" de cada área).
- convencoes/    Regras de idioma, código e git/deploy.
- adr/           Registro de decisões (o "porquê") com data e reversibilidade.
- roadmap.md     Ordem de implementação da base.

## Por onde começar
1. produto/visao-geral.md
2. arquitetura/visao-geral.md e arquitetura/stack.md
3. A EF do módulo que você vai tocar (EFs/).
4. convencoes/ e os ADRs relevantes antes de codar.

Regra de ouro do projeto: veja convencoes/codigo.md e arquitetura/seguranca.md.
